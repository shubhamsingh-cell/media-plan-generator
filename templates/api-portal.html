📖 Quick Start Guide

Generate an API Key

Go to the API Keys tab and create a new key. Choose the tier that matches your needs.

Include the Key in Requests

Add the X-API-Key header to all API requests. 

curl -X POST https://media-plan-generator.onrender.com/api/generate \
  -H "Content-Type: application/json" \
  -H "X-API-Key: mpg_your_key_here" \
  -d '{
    "company_name": "Acme Corp",
    "industry": "tech_engineering",
    "roles": "Software Engineer",
    "locations": "San Francisco, CA",
    "budget": 50000
  }'

Handle Responses

All responses are JSON. Check the status field: "success", "partial", or "error".

Monitor Usage

Check the Usage Dashboard tab to track your API calls, rate limits, and response times.

🔐 Authentication

All API requests require authentication via the X-API-Key header. API keys are prefixed with mpg_ and are hashed before storage for security. 

# Python example
import requests

response = requests.post(
    "https://media-plan-generator.onrender.com/api/generate",
    headers={
        "Content-Type": "application/json",
        "X-API-Key": "mpg_your_key_here"
    },
    json={
        "company_name": "Acme Corp",
        "industry": "tech_engineering",
        "roles": "Software Engineer",
        "locations": "San Francisco, CA",
        "budget": 50000
    }
)
print(response.json())

⚠ Rate Limits

Rate limits are enforced per API key on a daily basis (UTC). When exceeded, the API returns HTTP 429 with a JSON body containing the reset timestamp. 

# Rate limit exceeded response (HTTP 429)
{
    "error": "Rate limit exceeded",
    "limit": 100,
    "remaining": 0,
    "reset": "2026-03-23T00:00:00Z",
    "upgrade_url": "https://media-plan-generator.onrender.com/api-portal"
}

Rate limit headers are included in every response: X-RateLimit-Limit, X-RateLimit-Remaining, X-RateLimit-Reset.
